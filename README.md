# Классификация индийской еды: EfficientNet-B0 и ResNet18

В данном проекте решается задача многоклассовой классификации изображений 20 различных индийских блюд. Для решения используются две популярные архитектуры сверточных нейронных сетей — **EfficientNet-B0** и **ResNet18**, предобученные на ImageNet (Fine-tuning). Для интерпретации работы моделей и визуализации областей внимания применяется **Grad-CAM**.

## О датасете

Набор данных `Foodies_Challenge` включает изображения 20 классов индийских блюд.
* **Классы:** `Aloo_matar`, `Besan_cheela`, `Biryani`, `Chapathi`, `Chole_bature`, `Dahl`, `Dhokla`, `Dosa`, `Gulab_jamun`, `Idli`, `Jalebi`, `Kadai_paneer`, `Naan`, `Paani_puri`, `Pakoda`, `Pav_bhaji`, `Poha`, `Rolls`, `Samosa`, `Vada_pav`.
* **Обучающая выборка:** 9110 изображений
* **Тестовая выборка:** 2289 изображений

**Аугментация данных:** 
Для обучающей выборки применялись: `Resize`, `RandomHorizontalFlip`, `ColorJitter`. Для обеих выборок использовалась нормализация по стандартам ImageNet.

## Архитектура и обучение

Обе модели обучались с использованием следующих параметров:
* **Функция потерь:** CrossEntropyLoss
* **Оптимизатор:** Adam 
* **Размер батча:** 64
* **Устройство:** CUDA / CPU

## Результаты и сравнение моделей

| Модель | Accuracy (Test) | Weighted F1-score (Test) | Лучшая эпоха |
| :--- | :---: | :---: | :---: |
| **EfficientNet-B0** | **0.9205** | **0.9223** | **2-3** |
| **ResNet18** | 0.8471 | 0.8451 | 8 |

### Основные наблюдения:
1. **EfficientNet-B0 — победитель:** Эта архитектура показала значительно более высокие и стабильные результаты. 
2. **Проблема переобучения:** Обе модели склонны к переобучению. EfficientNet-B0 начинает переобучаться после 3-й эпохи, а обучение ResNet18 оказалось весьма нестабильным со скачками метрик на валидации.
3. **Сложные классы:**
   * Класс `Aloo_matar` показал наименьший F1-score из-за сильного дисбаланса (мало данных — всего 20 объектов в тесте).
   * Модели часто путают визуально похожие блюда: овощные рагу (`Kadai_paneer`, `Dahl`, `Pav_bhaji`) и лепешки (`Chapathi`, `Naan`).

## 👁️ Интерпретируемость (Grad-CAM)

Для понимания того, на какие части изображений опираются модели при принятии решений, был использован метод **Grad-CAM**.
* Обе сети в большинстве случаев корректно фокусируются на центральной части тарелки с едой.
* **EfficientNet-B0** выдает более сфокусированные и точные карты внимания по сравнению с ResNet18.
* В случаях ошибочной классификации карты внимания показывают, что модели смотрят на правильные области, но не могут уловить мелкие специфичные детали, необходимые для точного различия классов. Иногда внимание рассеивается на задний фон.

### Возможные улучшения

Для дальнейшего повышения качества моделей и борьбы с переобучением планируется:

* **Снизить Learning Rate** и внедрить планировщик (Learning Rate Scheduler).
* **Добавить регуляризацию** (например, Weight Decay) и Dropout.
* **Усилить аугментацию данных** (`RandomRotation`, `Cutout` и т.д.), чтобы улучшить обобщающую способность моделей.
* **Собрать больше данных** для малочисленных классов (таких как `Aloo_matar`).
